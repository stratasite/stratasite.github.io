## Skip the Setup

Every other tool makes you commit before you can ask anything: pick the right
semantic view, choose an explorer, open the correct cube, select a datasource. You have to know how
the model is carved up before you know what you want.

This step seems like a non issue when you first deploy a BI tool. But, you quickly end up 
with thousands of choices. Which explorer had that flavor of revenue you were looking at
last week?

Strata removes this step. Start with a rough idea. A measure or two you care
about and go straight into building. There is no construct to select first.

Furthermore, this ability to see the whole picture rather than a singular view or explorer
at once, turns out to be a game changer for Agentic use cases as well.  Agents are 
able to safely traverse the entire warehouse with great accuracy.

## Feedback as you Build

As you assemble a view, Strata validates and previews **per keystroke**. Add a
measure and you immediately see which dimensions it can be sliced by; add a
dimension and incompatible measures are filtered out before you can pick a wrong
combination. The answer takes shape live, so exploration is a conversation, not a
configure, run, debug cycle.

This is something no traditional BI tool does and it's why a vague idea is
enough to get started.

## Guided to Valid Answers

The same interactive loop that guides humans — validate a partial query, surface
the compatible next steps, reject invalid combinations with an explanation — is
structurally an **agent API**. Whether the builder is a person typing or an agent
calling, Strata keeps every step grain-safe and steers toward answers that are
both fast and correct.

## How it Works

Strata deploys a convention over configuration Semantic Modeling strategy where naming
is strict.  This has the dual benefit of creating the simplest semantic model protocol
while also enabling deep analytical self service.

Using names, field types, and table relationships, Strata constructs a hyper graph that enables 
extended relationship resolution. It was designed from day one to enable your non technical users to surf braodly across 
the data warehouse.
